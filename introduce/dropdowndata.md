# 📄 DropdownData.json - 介面選單選項設定檔

## 📝 檔案概述

`DropdownData.json` 是一個 **JSON 格式的設定檔**，用於定義 `main.py` 圖形介面中五個下拉選單（Dropdown）的可選內容。

此檔案將「選項數據」與「程式邏輯」分離，讓開發者或使用者**無需修改 Python 程式碼**，只需編輯此文字檔，即可新增或修改 AI 生成圖片時的提示詞參數。

## 🏗️ 資料結構 (Data Structure)

檔案內容為一個 **二維陣列 (List of Lists)**，結構如下：

```json
[
    [ "人物選項列表..." ],  // Index 0
    [ "著裝選項列表..." ],  // Index 1
    [ "配件選項列表..." ],  // Index 2
    [ "風格選項列表..." ],  // Index 3
    [ "情境選項列表..." ]   // Index 4
]

```

## 📊 內容詳細對照 (Content Mapping)

此檔案內的五個子列表分別對應 `main.py` 介面上的五個輸入欄位。系統會依照 **索引順序 (Index)** 讀取資料：

| 索引 (Index) | 對應變數 (main.py) | 介面標籤 | 用途說明 | 當前設定範例 |
| --- | --- | --- | --- | --- |
| **0** | `people` | **人物** | 定義畫面的主角身分 | 學生、老師、歌手、上班族 |
| **1** | `dress` | **著裝** | 定義人物穿著的服飾類型 | 西裝、禮服、古裝、冬季穿搭 |
| **2** | `accessories` | **配件** | 定義身上配戴的飾品 | 手串、玉珮、玉鐲 |
| **3** | `style` | **照片風格** | 定義 AI 生成圖片的藝術風格 | 動漫風、寫實風、墨水畫、雕刻 |
| **4** | `context` | **情境** | 定義人物的動作與背景故事 | 在山上打水、在茅屋歡呼 |

---

## 🔧 如何修改與擴充

若您希望在介面上增加新的選項（例如新增一種「賽博龐克」風格），請依照以下步驟操作：

1. 使用文字編輯器（如 Notepad, VS Code）開啟 `DropdownData.json`。
2. 找到對應的類別列表（例如 **第 4 個列表** 對應風格）。
3. 在列表中加入新的字串，並確保使用 **雙引號 (`"`)** 包裹，且項目之間用 **逗號 (`,`)** 分隔。

### 修改範例：新增「賽博龐克」風格

**修改前：**

```json
    [
        "動漫風",
        "電影風",
        ...
        "雕刻"
    ],

```

**修改後：**

```json
    [
        "動漫風",
        "電影風",
        ...
        "雕刻",
        "賽博龐克"  <-- 新增項目 (注意上一行結尾要補上逗號)
    ],

```

## 🔗 程式碼整合說明

在 `main.py` 中，這份檔案是透過以下程式碼片段被載入並應用：

```python
# 1. 讀取 JSON 檔案
with open('DropdownData.json', 'r', encoding='utf-8') as dropdown_data:
    DropDownList = json.load(dropdown_data)

# ... (略) ...

# 2. 將讀取到的列表分配給對應的 Dropdown 元件
# DropDownList[0] 對應 JSON 第一個列表
people = gr.Dropdown(label="人物", choices=DropDownList[0], ...)
dress  = gr.Dropdown(label="著裝", choices=DropDownList[1], ...)
# ... 以此類推

```

## ⚠️ 注意事項

1. **JSON 格式嚴格**：字串必須用雙引號 `"`，不能用單引號 `'`。
2. **列表順序**：請勿隨意調換五個大列表的順序，否則會導致「人物」選單裡出現「風格」選項的情況。
3. **編碼**：存檔時請務必使用 **UTF-8** 編碼，以免中文顯示亂碼。
